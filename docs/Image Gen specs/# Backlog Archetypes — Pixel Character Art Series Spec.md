# # Backlog Archetypes — Pixel Character Art Series Spec

## Project Overview

This project is a series of **distinct archetype character illustrations** representing different gamer/backlog behaviors.

The goal is to create a set of character artworks that:

- feel like they belong in the **same app / product ecosystem**
- feel like they were made by the **same studio / same artist brain**
- but allow each archetype to have a **wildly different vibe, silhouette, species, personality, fashion language, and mood**

These are not generic avatars.  
They are **characterful hero illustrations** with strong identity.

---

## Core Intent

Each archetype should feel like:

- a **recognizable personality**
- a **fun visual metaphor**
- a **character worth collecting**
- something that could sit inside an app, achievement system, or game-like interface

This is not just “a gamer standing there.”
Each one should communicate the archetype immediately through:

- pose
- species/body type
- wardrobe
- props
- orbiting icons/glyphs
- expression
- energy
- visual storytelling

---

## Output Modes

### Default Mode (current preferred mode)
**Hero Character / Transparent Background**

Create:
- a single full-body or near-full-body character
- on a **transparent background**
- no full room / no environmental scene
- no built-out foreground/background set dressing by default

Allowed:
- held objects
- worn accessories
- floating icons
- small orbiting props
- small familiar/mascot companions if fitting
- UI glyphs / stat panels / symbolic archetype indicators

This should feel like a **premium PNG hero asset**.

---

### Secondary Mode (optional later)
**Expanded Scene Version**

A more fleshed-out scene-rich illustration may be created later for some archetypes:
- background/room/environment
- CRT TVs, shelves, cats, clutter, etc.
- stronger environmental storytelling

This is **not the default** right now.

---

## Visual Style

### Core Style Target
- **High-detail pixel art**
- rich, polished, premium-looking
- expressive and character-driven
- cozy / gamey / collectible / stylized
- clean silhouette and clear read at a glance
- detailed enough to reward zooming in

### Style Cohesion
Across the whole series, the art should feel like:
- “same studio”
- “same general visual family”
- “same product ecosystem”
- but **different design thinking per character**

### Style Flexibility
The style may drift slightly depending on the archetype:
- can lean a bit anime
- can lean a bit Ghibli-ish
- can lean whimsical, sci-fi, cyberpunk, folkloric, soft fantasy, etc.

But it should **not** drift so far that it feels like a completely different franchise.

---

## Series-Level Principles

1. **Each archetype should have a distinct vibe**
   - not just palette-swapped versions of the same person
   - not just “same room, same character, different props”

2. **Each archetype can be a different being**
   Allowed examples:
   - human
   - gnome
   - cyclops
   - robot
   - cyborg
   - holographic plasma entity
   - wise old man
   - streamer / influencer
   - time traveler
   - fairy / goblin / archivist / android / etc.

3. **Different archetypes should contrast strongly**
   Example:
   - one can feel street-punk / nerd / backlog goblin
   - another can feel serene and elegant
   - another can feel ancient and mystical
   - another can feel hyper-modern and neon
   - another can feel soft and cozy
   - another can feel sterile and optimized
   - another can feel regal and composed

4. **Not every character should be cluttered**
   Important rule:
   - some archetypes benefit from a lot of props, badges, charms, miscellania
   - some archetypes should be simpler, cleaner, calmer, or more iconic

   Example:
   - one archetype may have overloaded bags and dangling objects
   - another may work better with a silky robe and a single amulet
   - another may need almost no ornamentation

5. **Clutter is a tool, not a mandate**
   Use density only when it supports the archetype.

---

## Composition Spec

### Preferred Composition
- one character
- hero framing
- full-body or almost full-body
- centered or slightly dynamic off-center
- transparent background
- readable silhouette
- enough whitespace around the character for UI use / app use

### Allowed Additions
- floating icons
- glyphs
- tiny UI panels
- orbiting symbols
- a few props in motion
- companion/familiar if fitting
- archetype metrics rendered as stylized floating UI

### Avoid
- full room scenes by default
- grounded environmental background
- large furniture
- elaborate floor planes
- overbuilt scenery that turns it into an illustration instead of a hero asset

---

## Resolution / Deliverable Intent

The asset should feel like:
- a **hero image**
- suitable for PNG export
- transparent background
- high resolution
- polished enough for real product/interface usage

---

## Character Design Rules

Each character should communicate the archetype through some combination of:

### 1. Silhouette
- tall / tiny / wide / elegant / bulky / angular / floaty / geometric / monstrous / regal / goblin-like, etc.

### 2. Body language
- confident
- overwhelmed
- distracted
- serene
- intense
- smug
- methodical
- exhausted
- curious
- chaotic

### 3. Props
Use props that make the archetype instantly legible.

Examples:
- backlog receipts
- save files
- sealed games
- tiny genre tokens
- wishlists
- progress bars
- trophies
- sale tags
- sorting tools
- notebooks
- seeds
- old cartridges
- completion crowns
- timers
- reviews/ratings
- achievement panels
- installed-file cubes
- platform emblems

### 4. Apparel
Wardrobe should reinforce the archetype.

Examples:
- archivist
- streetwear
- druidic
- royal
- utilitarian
- influencer
- sci-fi pilot
- monk
- retro-tech collector
- fantasy wizard
- minimalist perfectionist
- washed-out insomniac
- bargain-bin treasure hunter

### 5. Species / form factor
Species/form can be used as metaphor:
- gnome = playful sampler / dabbler
- robot = optimizer / system thinker
- cyclops = focused sniper
- wise elder = archaeologist
- hologram = infinite / non-finishable game spirit
- etc.

---

## Orbiting Icons / Glyphs / UI Elements

These are encouraged when useful.

They should function like:
- gameplay affordances
- symbolic shorthand
- humorous stat callouts
- genre signals
- platform hints
- achievement/UI flavor

Examples:
- progress bars
- “unfinished” tags
- save-file panels
- floating genre icons
- completion %
- start vs finish counters
- platform badges
- discount tags
- review score meters
- wishlist stars
- little coins / boxes / crates / seeds / trophies / energy bars

These should feel integrated with the character, not random decoration.

---

## Text Usage

Short readable text is okay and often useful.

Examples:
- “BACKLOG: 999+”
- “COMPLETED: 3%”
- “NEW GAME?”
- “STARTS: 42 / FINISHES: 2”
- “I’LL PLAY IT SOMEDAY”
- “SAVE 01”
- “UNFINISHED”
- “LATER”

### Text Guidance
- short
- clear
- witty
- game-like
- legible
- supportive of the joke / archetype

Avoid:
- giant paragraphs
- too much copy
- text overload that becomes messy unless the archetype specifically benefits from it

---

## Tone by Archetype

The tone field matters.

### Roast
Use affectionate mockery.
The art can tease the behavior but should still be lovable.

### Respect
Make them aspirational, admirable, or heroic.

### Neutral
Represent clearly without over-roasting or glorifying.

---

## Series Cohesion Rules

Even though vibes differ, the set should still cohere via:

- polished pixel-art rendering
- strong character-first composition
- similar overall production quality
- similar “ecosystem” thinking
- recurring use of small UI glyphs or symbolic objects when appropriate
- shared sensibility: playful, smart, collectible, expressive

Think:
> “A family of very different archetypes, not clones.”

---

## Important Negative Guidance

Do **not** default to:
- the same room
- the same body type
- the same face
- the same coat/jacket
- the same gamer setup
- the same prop language
- the same clutter density
- the same emotional tone

Do **not** make every character:
- street punk
- anime teen
- backpack hoarder
- collector in a room
- maximalist by default

Variation is essential.

## Creative Freedom Rules

When generating a new archetype, the artist/model is encouraged to ask:

- What species or being best expresses this behavior?
- What silhouette makes the archetype instantly recognizable?
- Should this one be cluttered or minimal?
- Should this feel soft, regal, anxious, futuristic, mystical, comic, elegant, or absurd?
- What 3–6 props best explain the archetype?
- What small orbiting icons would make the concept clearer?
- What clothing vocabulary best expresses the behavior?

---

## Default Prompting Logic for Future Archetypes

When generating the next character:
1. Use the same overall polished pixel-art ecosystem style
2. Make the new character feel clearly distinct from the previous ones
3. Use transparent background
4. Focus on character-first hero asset
5. Use floating icons / small props if helpful
6. Avoid scene-building unless explicitly requested
7. Use clutter only if appropriate
8. Preserve product-family cohesion while changing vibe dramatically
