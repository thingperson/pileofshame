# DALL-E Theme Asset Prompts

Visual boost prompts for every theme in Inventory Full. Each theme section includes prompts for: header/banner, background texture, icons/status set, footer, and any theme-specific special elements.

**Batch workflow tip:** DALL-E can process these one at a time. For efficiency, group by asset type (do all headers first, then all backgrounds, etc.) so you stay in the same creative headspace. Use "in the style of [previous image]" to maintain consistency within a theme.

**Technical specs:**
- Headers/banners: 1200×200px, wide format
- Backgrounds: 1024×1024px, seamless/tileable preferred
- Icons: 256×256px each, transparent background when possible
- Footers: 1200×150px, wide format
- Hero/feature images: 1536×1024px

## Batch Prompting Strategy

For maximum efficiency in DALL-E:

**Round 1 — Backgrounds (all themes):**
Run all background texture prompts. These are the easiest to generate and most consistent.

**Round 2 — Headers (themed themes only):**
Skip Dark, Light, Void, Minimal. Do: 80s, 90s, Future, Dino, Weird, Ultra, Cozy, Tropical, Campfire.

**Round 3 — Icon sets:**
Do Dino and Tropical first (most distinct). Then Campfire, 80s, Dark.

**Round 4 — Footers + special elements:**
These are nice-to-haves. Prioritize: Dino footer, Campfire footer, Cozy footer.

**Round 5 — Landing page:**
Hero background and section dividers. One-time assets.

**Round 6 — Platform themes:**
Only if we decide to build these. PlayStation and Xbox first.

## Integration Notes

Once assets are generated:
1. Save to `public/themes/[theme-name]/` directory structure
2. Reference in CSS as `background-image: url('/themes/[theme]/bg.png')`
3. Headers go into theme-specific banner components in NinetiesMode.tsx
4. Icons can replace emoji in STATUS_CONFIG (constants.ts)
5. Run through image optimization (squoosh.app or sharp) before committing — keep PNGs under 200KB each
